# Витрина студенческих проектов СевГУ

Веб-платформа для публикации, поиска и совместной работы над студенческими проектами Севастопольского государственного университета.

## Функциональность

### Для пользователей:
- **Публикация проектов** - создание и публикация студенческих проектов с подробной информацией
- **Банк задач** - публикация задач для решения студенческими командами
- **Командообразование** - создание команд, открытие вакансий и поиск участников
- **Поиск и фильтрация** - удобный поиск проектов по различным критериям
- **Профили пользователей** - информация о студентах и их достижениях

### Для администраторов:
- **Управление системой** - полный контроль над платформой
- **Аналитика** - детальная статистика по проектам, институтам, технологиям
- **Управление пользователями** - просмотр и управление пользователями
- **Модерация контента** - контроль над публикуемыми проектами и задачами

## Технологии

- **Backend**: Laravel 11, PHP 8.2+
- **Frontend**: Bootstrap 5, JavaScript, Font Awesome
- **База данных**: SQLite (для разработки)
- **Аутентификация**: Laravel UI с Bootstrap

## Требования

### Обязательные компоненты:
- **XAMPP** - для работы с PHP, Apache и MySQL
- **PHP 8.2+** (входит в XAMPP)
- **Composer** - менеджер зависимостей PHP
- **Node.js и npm** - для сборки фронтенда

### Установка XAMPP:
1. Скачайте XAMPP с официального сайта: https://www.apachefriends.org/
2. Установите XAMPP в папку `C:\xampp\`
3. Запустите XAMPP Control Panel
4. Включите Apache и MySQL (если используете MySQL вместо SQLite)

## Установка

### Шаги установки

1. **Клонирование репозитория**
```bash
git clone <repository-url>
cd ProjectFair
```

2. **Установка зависимостей**
```bash
composer install
npm install
```

3. **Настройка окружения**
```bash
cp .env.example .env
php artisan key:generate
```

4. **Настройка базы данных**
Отредактируйте файл `.env`:
```env
DB_CONNECTION=sqlite
DB_DATABASE=/absolute/path/to/database.sqlite
```

5. **Создание базы данных**
```bash
touch database/database.sqlite
```

6. **Запуск миграций**
```bash
php artisan migrate
```

7. **Добавление тестовых данных**
Выполните SQL-файл `database_insert.sql` в вашей СУБД или используйте:
```bash
php artisan db:seed
```

8. **Создание символической ссылки для файлов**
```bash
php artisan storage:link
```

9. **Сборка фронтенда**
```bash
npm run dev
```

10. **Запуск сервера**
```bash
php artisan serve
```

### Альтернативный способ добавления тестовых данных

Если вы хотите использовать готовый SQL-файл с тестовыми данными:

1. Выполните миграции: `php artisan migrate`
2. Откройте файл `database_insert.sql` в вашем SQL-клиенте
3. Выполните все SQL-запросы из файла
4. Данные будут добавлены в базу данных

**Примечание**: Все пароли в тестовых данных - `password`

## API

Система использует стандартные RESTful маршруты Laravel:

- `GET /projects` - список проектов
- `GET /projects/{id}` - просмотр проекта
- `POST /projects` - создание проекта (требует аутентификации)
- `PUT /projects/{id}` - обновление проекта
- `DELETE /projects/{id}` - удаление проекта
